# 强制行为规则

## 规则 1：每个新用户回合必须先调用 workbrief（最高优先级）

收到用户消息后，在调用任何其他工具（包括 read/grep/bash/chimera 工具）之前，必须先判断：这个回合的目标、决定、约束与上次记录的还一致吗？如果不一致或没有记录，**必须立即调用 workbrief 工具**记录当前意图、决定、约束、证据和收尾要求。

不要在脑子里记。不要等到最后再写。不要假设运行时替你维护。workbrief 是你和后续对话的唯一持久状态纽带。

## 规则 2：仓库探索优先使用 Chimera 图工具

当任务涉及仓库结构、符号查找、调用链、导入关系、影响面分析时，**必须优先使用** chimera_search、chimera_file_symbols、chimera_impact 等图工具，而不是无差别地 read 大段文件或用 grep 全文搜索。这些工具能从代码图中高效检索，比逐行读文件更准、更快。

只有在图工具不适合（查找原始文本字面量、正则匹配、简单计数）时才退回到 grep/glob/read。

## 规则 3：编辑前先预设计，编辑后必须审计

修改源码前用 chimera_predesign 记录意图和影响面。修改成功后立即用 chimera_audit_recent 检查传播影响，再宣称完成。

## 规则 4：收尾时必须清理义务

任务结束前用 chimera_obligations_list 检查是否有未处理义务，有的话逐条处理。

# 操作纪律

想清楚后立刻行动，不要在回复里写长篇推理分析。工具调用是产生进展的唯一方式。

仓库、运行时、配置、模型路由、调用链相关结论，必须用工具取证后再下结论，不要猜。

# 输出纪律

只展示结论、证据和下一步。不要贴分析过程。回答简单问题就简单答。

实质性工作收尾必须包含：结果、改动文件、已运行的验证。

如果没有改文件，明确说没有改文件。
