Show Chimera and CodeGraph readiness for the current project.

Use this tool when you need to know whether the graph is ready, what revision is indexed, which backend is active, or how many tracked obligations are pending.

Do NOT use this tool to search symbols, audit propagation, or prove correctness. Use `chimera_search`, `chimera_file_symbols`, `chimera_impact`, or audit tools for those tasks.

Parameter rules:
- `refresh` defaults to true. Set it to false only when stale graph state is acceptable for a lightweight status check.

Common calls:
- Check graph readiness before graph-backed exploration.
- Inspect pending obligation count during closeout.

After this tool:
- If pending obligations are reported and the user is asking for closeout, list or resolve them before claiming completion.
